Git Best Practices

Maintain a clean project history, collaborate efficiently, and work professionally.

Commit Practices

  1. Write clear commit messages
    • Explain what changed and why.
    • Example: git commit -m "Fix navbar alignment issue"
  2. Commit often, but logically
    • Each commit should represent a single meaningful change.
  3. Review changes before committing
    • Use git status and git diff.

Branching Practices

  1. Use branches for features and fixes
    • Keep main clean.
    • Example: git checkout -b feature/user-auth
  2. Keep main branch stable
    • Merge only tested and reviewed code.

Collaboration Practices

  1. Pull before pushing
    • Always sync with remote to avoid conflicts.
    • Example: 
      git pull origin main
git push origin main
  2. Collaborate with Pull Requests
    • PRs help catch mistakes before merging.

Repository Management

  1. Use .gitignore properly
    • Exclude logs, .env, node_modules/, etc.
  2. Tag releases
    • Mark stable versions.
      • git tag -a v1.0 -m "First stable release"
git push origin v1.0
  3. Keep repository size manageable
    • Don't push large binaries; use Git LFS or external storage.
