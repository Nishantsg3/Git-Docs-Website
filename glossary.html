Git Glossary

Quick reference for common Git terms and commands.

Common Git Terms

Commit
A snapshot of changes in your repository at a specific point in time.
Branch
An independent line of development, allowing separate work on features or fixes.
Merge
Combining changes from one branch into another.
Clone
Creating a copy of a repository from a remote source to your local machine.
Pull
Fetching and integrating changes from a remote repository to your local branch.
Push
Uploading your local commits to a remote repository.
Rebase
Moving or combining a sequence of commits to a new base commit.
Tag
Marking a specific commit with a human-readable label, often for releases.
.gitignore
Specifies which files or directories Git should ignore.
Stash
Temporarily saving changes that are not ready to commit so you can work on something else.
