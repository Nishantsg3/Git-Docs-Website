Git & GitHub Docs

A complete guide from Beginner → Intermediate → Advanced, with best practices and shortcuts.

Introduction

Git is a distributed version control system that helps developers track changes in code over time. It allows you to manage projects efficiently, collaborate with others, and maintain a history of all modifications.

GitHub is a cloud-based hosting platform that uses Git. It provides tools for collaboration, pull requests, code reviews, and project management.

Together, Git and GitHub are essential skills for every developer, from beginners to professionals.

Roadmap

This documentation is divided into levels so you can learn Git & GitHub step by step:

  1. Beginner - Installing Git, first repository, commits, pushing to GitHub.
  2. Intermediate - Branching, merging, pull requests, resolving conflicts.
  3. Advanced - Rebasing, stashing, workflows, CI/CD integration.
  4. Best Practices - Smart commit messages, aliases, `.gitignore`, collaboration tips.
  5. Glossary - Quick reference of Git & GitHub terms.

Quick Reference

Repository (repo)
A storage location where your project's files and history are tracked using Git.
Commit
A snapshot of changes made to the code, saved in the repository history.
Branch
A separate line of development that allows you to work on new features without affecting the main code.
Merge
The process of combining changes from one branch to another.
Push
Uploads your local commits to a remote repository (like GitHub).
