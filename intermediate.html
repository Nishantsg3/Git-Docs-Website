Git Intermediate

Branching, merging, pull requests, and resolving conflicts – take your Git skills to the next level.

Branching

Create separate lines of development to work independently:

git branch feature-branch
git checkout feature-branch

Merging

Combine changes from one branch into another:

git checkout main
git merge feature-branch

Resolve any merge conflicts manually if needed.

Pull Requests

Push your feature branch to GitHub and create a Pull Request (PR) for code review and discussion:

git push origin feature-branch

Then, open a PR on GitHub and request review before merging.

Resolving Conflicts

Sometimes changes clash when merging. Use:

git status
git add <resolved-file>
git commit

This stages and commits the resolved changes.
